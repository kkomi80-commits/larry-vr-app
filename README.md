## 📱 설치 방법

### GitHub Pages로 배포

#### 1단계: GitHub 저장소 생성
```bash
1. GitHub.com 로그인
2. "New repository" 클릭
3. 이름: larry-vr-pro
4. Public 선택
5. "Create repository" 클릭
```

#### 2단계: 파일 업로드
```
다음 3개 파일을 저장소에 업로드:
- index.html
- manifest.json
- sw.js
```

#### 3단계: GitHub Pages 활성화
```bash
1. 저장소 Settings 이동
2. 왼쪽 메뉴에서 "Pages" 클릭
3. Source: "Deploy from a branch"
4. Branch: "main" 선택
5. Save 클릭
6. 5분 후 URL 확인 (예: https://yourname.github.io/larry-vr-pro/)
```

### 아이폰에 설치

#### Safari에서:
```
1. 생성된 URL 접속
2. 하단 공유 버튼 (□↑) 탭
3. "홈 화면에 추가" 선택
4. "추가" 탭
5. 완료! 📱
```

## 🚀 빠른 시작 가이드

### 1️⃣ 첫 실행 (30초 설정)

#### A. VR 기본 설정
```
풀(Pool) 현금: 보유 현금 입력 (예: 10000)
G값: 기본 10 (보수적: 15-20, 공격적: 5-8)
```

#### B. 보유 주식 입력
```
SOLX: 보유 수량 입력
TQQQ: 보유 수량 입력
```

#### C. 목표 비율 설정
```
SOLX: 50% (기본값)
TQQQ: 50% (기본값)
또는 원하는 비율로 조정
```

### 2️⃣ 주가 업데이트 (2가지 방법)

#### 방법 1: API 자동 업데이트 (권장)
```
1. "⚙️ 설정" 버튼 클릭
2. 또는 "API 키 설정하기" 클릭
3. alphavantage.co에서 무료 API 키 발급
4. API 키 입력 후 저장
5. "API 업데이트" 버튼 클릭
```

#### 방법 2: 수동 입력
```
1. "수동 입력" 버튼 클릭
2. 각 주식의 현재가 직접 입력
3. "입력 완료" 클릭
```

### 3️⃣ 리밸런싱 확인
```
✅ V 비율 자동 계산
✅ 임계값(±5%) 벗어나면 빨간색 표시
✅ 정확한 매수/매도 수량 제시
```

## 💡 주요 기능 상세 설명

### 1. 실시간 주가 섹션
```
📊 현재가 표시
📈 보유 수량 확인
⚡ API/수동 업데이트
🕐 마지막 업데이트 시간
```

**자동 새로고침:**
- 설정에서 활성화
- 1분마다 자동 업데이트
- API 제한 주의!

### 2. VR 시스템
```
공식: V = P + (P / G)

V: 목표 자산 가치
P: 풀(예수금)
G: 기울기
```

**G값 가이드:**
- G = 5~8: 공격적 (빠른 성장)
- G = 10: 중립적 (균형)
- G = 15~20: 보수적 (안정 중시)

### 3. 리밸런싱 알림
```
🔔 V 비율이 임계값 벗어나면 알림
⏰ 1시간마다 체크
📱 브라우저 푸시 알림 (선택)
```

**알림 설정:**
1. 설정 → "리밸런싱 알림" 활성화
2. 브라우저 권한 허용
3. 임계값 조정 (기본 5%)

### 4. 투자 기록
```
💾 원클릭 저장
📊 최대 50개 기록 보관
📈 시점별 V 비율 추적
📉 포트폴리오 변화 분석
```

### 5. 데이터 백업
```
📥 JSON 파일로 내보내기
💾 로컬 저장소 백업
🔄 다른 기기로 이동 가능
```

## ⚙️ 설정 옵션

### 자동 새로고침
```
ON: 1분마다 자동 가격 업데이트
OFF: 수동 버튼 클릭시만 업데이트

⚠️ 주의: API 일일 제한 (500회)
💡 팁: 장중에만 켜기
```

### 리밸런싱 알림
```
ON: V 비율 이탈시 자동 알림
OFF: 알림 없음

⚠️ 브라우저 알림 권한 필요
```

### 임계값 설정
```
기본: 5%
범위: 1~20%

낮을수록: 민감한 리밸런싱
높을수록: 여유로운 리밸런싱
```
